api documentation

the bronx api allows you to programmatically access statistics, economy data, and bot metadata. our api follows rest principles and returns json responses.

1. introduction

the api is divided into public endpoints (accessible by anyone) and authenticated endpoints (requiring a discord session). all requests should be made to https://bronxbot.xyz/api/.

2. authentication

authentication is handled via discord oauth2. when you log into the bronx dashboard, a session cookie is established. this cookie must be present in the headers for all non-public requests.

csrf protection

for state-changing requests (post, patch, delete), you must provide a x-csrf-token header. you can fetch the current token from /api/csrf-token.

3. public endpoints

get /health public

check the status of the bot, database, and cache systems.

{ "status": "healthy", "database": "connected", "cache": "redis", "timestamp": "2026-04-16T15:00:00Z" }
get /leaderboard/global public

fetch the global top 100 players by economy balance.

4. dashboard api

get /auth/user public/auth

returns the current authenticated user's discord profile.

get /guilds authenticated

returns a list of discord servers where both the user is an administrator and the bot is present.

5. bot metadata

get /guide public

returns categorical documentation for all available bot commands.

get /bot/log public

returns recent public bot events (guild joins, major updates).
